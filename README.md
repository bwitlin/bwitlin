### 🛠 What I'm building right now

**[claude-skills](https://github.com/bwitlin/claude-skills)** -- an opinionated, growing kit of Claude Code skills and rules for people who already have a `CLAUDE.md` and want their AI to actually behave like a senior teammate.

Most "Claude templates" tell the model *what* to do. This repo fixes how it *thinks* -- sycophancy, false confidence, execution-mode drift, skill routing, and the other quiet failure modes that turn a great agent into a forgettable one.

**Who it's for:** intermediate Claude Code users who've outgrown starter templates and want *behavioral* upgrades, not more boilerplate.
**Why care:** the difference between a useful agent and a magical one isn't more context -- it's better instincts. This repo is my attempt to install instincts.

#### 🎯 Skills

- **[`/feedback`](https://github.com/bwitlin/claude-skills/tree/main/plugins/feedback)** -- A workflow mirror. 5/5/5 reports that track what's improving, escalate what isn't, and turn repeat advice into automation.
- **[`/skill-battle`](https://github.com/bwitlin/claude-skills/tree/main/plugins/skill-battle)** -- A cage match for ideas -- run multiple skills on the same task in parallel and compare outputs side by side.
- **[`/eli5`](https://github.com/bwitlin/claude-skills/tree/main/plugins/eli5)** -- Simple, visual, concrete explanations for adults who want the aha moment without the jargon. Research-backed, opens in your browser.

#### 📐 Rules

- **[`active-context-header`](https://github.com/bwitlin/claude-skills/tree/main/rules/active-context-header)** -- Who? Where? What? -- on every response. Active skill, current project, and one of five task states: in progress, completed-and-next, pivoting, waiting on you, or blocked.
- **[`anti-sycophancy`](https://github.com/bwitlin/claude-skills/tree/main/rules/anti-sycophancy)** -- Cures Claude's overly agreeable caving problem. Defend the recommendation with evidence when pushed back, or name the new data that changed your mind.
- **[`skill-dispatch-protocol`](https://github.com/bwitlin/claude-skills/tree/main/rules/skill-dispatch-protocol)** -- Three-tier routing for which skill fires when your message could match more than one. Auto-invoke when obvious, present options when ambiguous, confirm before anything destructive.
- **[`secret-file-protection`](https://github.com/bwitlin/claude-skills/tree/main/rules/secret-file-protection)** -- Blocks reading .env, credentials, keys, and shell config files that contain secrets.
- **[`confidence-signaling`](https://github.com/bwitlin/claude-skills/tree/main/rules/confidence-signaling)** -- Three confidence labels ("I know" / "I think" / "I'm not sure") plus an evidence-strength grade (strong / medium / weak). You always know which claims to trust and how solid the data behind them is.
- **[`execution-mode`](https://github.com/bwitlin/claude-skills/tree/main/rules/execution-mode)** -- For humans with limited context windows. One action per response, no lectures, no education -- just the next thing to do.
